# نظام إدارة المخزون المبسط (Simple Inventory Management System)

نظام ويب متكامل لإدارة المخزون والمبيعات والمشتريات، مبني بلغة **PHP** (نمط إجرائي) مع قاعدة بيانات **MySQL / MariaDB**، وواجهة عربية بالكامل (RTL).

---

## 📋 نظرة عامة

يغطي النظام دورة عمل متجر أو محل تجاري بشكل شبه كامل: من تسجيل المنتجات والفئات، مروراً بالشراء من الموردين والبيع للعملاء، وصولاً إلى تحصيل المدفوعات وتتبع حركة المخزون تلقائياً.

## ✨ المزايا الرئيسية

- **تسجيل دخول وصلاحيات**: نظام مستخدمين بدورين (`admin` و `merchant`) مع حماية الجلسات (Sessions).
- **إدارة المنتجات**: إضافة/تعديل/حذف مع ربط بالفئة والمورد، وتنبيه عند انخفاض المخزون عن الحد الأدنى.
- **إدارة الفئات والموردين**: تصنيف المنتجات وربطها بجهات التوريد.
- **إدارة العملاء**: سجل عملاء يمكن ربطه بفواتير البيع.
- **المشتريات**: تسجيل الشراء من المورد مع تحديث المخزون تلقائياً (زيادة) وتسجيل حركة "وارد".
- **المبيعات**: تسجيل البيع للعميل مع التحقق من توفر الكمية، وخصمها تلقائياً من المخزون، وتسجيل حركة "صادر".
- **المدفوعات**: تحصيل قيمة الفواتير بشكل كامل أو جزئي أو آجل، مع تحديث تلقائي لحالة الفاتورة (مدفوعة / جزئي / غير مدفوعة).
- **حركة المخزون**: سجل تتبع كامل لكل عملية إضافة أو سحب أو تعديل على المخزون.
- **إدارة المستخدمين**: (للأدمن فقط) إضافة/تعديل/حذف حسابات الدخول.
- **لوحة تحكم إحصائية**: عدد المنتجات، قيمة المخزون، المنتجات منخفضة المخزون، عدد العملاء، إجمالي المبيعات، والمبالغ المستحقة على العملاء.

## 🗂️ هيكل قاعدة البيانات

| الجدول | الوصف |
|---|---|
| `users` | حسابات الدخول (admin / merchant) |
| `categories` | فئات المنتجات |
| `suppliers` | الموردون |
| `products` | المنتجات (مرتبطة بفئة ومورد) |
| `stock_movements` | سجل حركة المخزون (وارد / صادر / تعديل) |
| `customers` | العملاء |
| `purchases` | عمليات الشراء من الموردين |
| `sales` | عمليات البيع للعملاء |
| `payments` | المدفوعات المحصّلة على فواتير البيع |

### أبرز العلاقات (Foreign Keys)

```
purchases.product_id   → products.id
purchases.supplier_id  → suppliers.id
sales.product_id       → products.id
sales.customer_id      → customers.id
payments.sale_id       → sales.id   (ON DELETE CASCADE)
```

## ⚙️ السلوك الآلي المرتبط بالمخزون

- **عملية شراء جديدة** → زيادة `stock_quantity` تلقائياً + تسجيل حركة "وارد".
- **عملية بيع جديدة** → التحقق من توفر الكمية أولاً، ثم خصمها تلقائياً + تسجيل حركة "صادر". إذا كانت حالة الدفع "مدفوعة بالكامل" تُسجَّل دفعة تلقائية بكامل القيمة.
- **تعديل/حذف عملية شراء أو بيع** → إعادة ضبط المخزون تلقائياً بالفرق (أو بالكامل عند الحذف).
- **إضافة/تعديل دفعة** → تحديث حالة الفاتورة تلقائياً (مدفوعة / جزئي / غير مدفوعة)، مع منع إدخال مبلغ يتجاوز المتبقي على الفاتورة.

## 🛠️ التقنيات المستخدمة

- **Backend**: PHP (mysqli, إجرائي بالكامل)
- **قاعدة البيانات**: MySQL / MariaDB
- **Frontend**: HTML/CSS مضمّن داخل ملفات PHP، خط Tajawal، أيقونات Font Awesome
- **الجلسات والمصادقة**: PHP Sessions + `password_hash`/`password_verify` (bcrypt)

## 📦 متطلبات التشغيل

- PHP 7.4 أو أحدث (يدعم `mysqli`)
- MySQL أو MariaDB
- خادم ويب (Apache/Nginx) أو خادم PHP المدمج للتطوير المحلي

## 🚀 خطوات التثبيت والتشغيل محلياً

1. إنشاء قاعدة البيانات عن طريق استيراد ملف `inventory_db.sql`:
   ```bash
   mysql -u root -p < inventory_db.sql
   ```
2. التأكد من إعدادات الاتصال في `config.php` (اسم المستخدم، كلمة المرور، اسم القاعدة `simple_inventory_db`).
3. تشغيل خادم PHP المحلي من داخل مجلد المشروع:
   ```bash
   php -S localhost:8000
   ```
4. فتح المتصفح على `http://localhost:8000/login.php`.

### حسابات دخول افتراضية

| اسم المستخدم | الدور | كلمة المرور |
|---|---|---|
| `admin` | مدير النظام | `admin123` |
| `admin2` | مدير إضافي | `admin123` |
| `tajer1` | تاجر (merchant) | `admin123` |

> ⚠️ يجب تغيير كلمات المرور الافتراضية فوراً قبل أي استخدام فعلي أو نشر للنظام.

## 🔒 ملاحظات أمنية مهمة

النظام في وضعه الحالي **مناسب للتعلّم والتجربة المحلية**، لكنه يحتاج تعديلات قبل أي استخدام إنتاجي حقيقي، أبرزها:

- استخدام **Prepared Statements** بدل الدمج المباشر للمتغيرات داخل جمل SQL (النظام حالياً يعتمد على `mysqli_real_escape_string` والـ casting كحماية أساسية من SQL Injection).
- تفعيل **HTTPS** وحماية إضافية للجلسات (session).
- عدم ترك كلمة مرور قاعدة البيانات فارغة في `config.php` في بيئة الإنتاج.
- إضافة حماية CSRF على النماذج (Forms).
